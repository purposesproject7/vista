# Faculty Dashboard Tutorial - User Guide

## Getting Started

Welcome to the VISTA Faculty Dashboard Tutorial! This comprehensive guide will teach you how to effectively use all features of the faculty dashboard for managing student project reviews and assessments.

## How to Access the Tutorial

1. **Login** to your faculty account
2. **Click your profile** in the top-right corner
3. **Select "Walkthrough/Tutorial"** from the dropdown menu
4. The tutorial will open in a new page

## Learning Path

### For New Faculty (30-45 minutes)
1. Start with the **Welcome** section to understand the overview
2. Focus heavily on the **Filter System** - this is crucial
3. Spend time understanding **Statistics** and **Active Reviews**
4. Practice with the **interactive elements**
5. Review **Assessment Best Practices**

### For Experienced Faculty (15-20 minutes)
1. Quick review of **Filter System** updates
2. Focus on **Assessment Workflow** improvements
3. Check **Overdue Reviews** handling procedures
4. Review **New Features** if any

### For Troubleshooting (10-15 minutes)
1. Go directly to the relevant section

4. Reference **Best Practices** sections

## Key Conditions & Requirements

### ‚ö†Ô∏è Critical Requirements
- **All Filters Must Be Selected**: You cannot access reviews without completing Year, School, Programme, and Type
- **Role-Based Access**: You can only see reviews for your assigned schools/programmes
- **Deadline Awareness**: Overdue reviews (orange) must be completed immediately
- **Individual Assessment**: Each team member must be marked individually

### üîí Security Requirements
- **Regular Password Updates**: Change password every 90 days
- **Secure Logout**: Always logout on shared computers
- **No Credential Sharing**: Never share login information

### üìù Assessment Requirements
- **Rubric Compliance**: Use provided performance levels
- **Detailed Feedback**: Written comments are mandatory
- **Submission Finality**: Marks cannot be changed without special approval
- **Deadline Compliance**: Submit before or on deadline when possible
